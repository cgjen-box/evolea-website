<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → /api/csp-report | Untrusted POST bodies (CSP report JSON) cross into the SSR worker |
| browser → all routes | Response headers are the security control surface; misconfiguration can break Keystatic OAuth or Formspree |
| src/lib/security-headers.ts → public/_headers | Two emission paths for the same policy; divergence is the named failure mode |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Denial of Service | /api/csp-report | mitigate | Endpoint reads body in try/catch, never throws, always returns 204; no storage/processing of report payload (no amplification). Report-Only mode limits report volume. No logging by default (no log-injection sink). |
| T-01-02 | Tampering | CSP report body / log injection | mitigate | Report body is never logged or reflected; returned 204 has empty body, so attacker-controlled content never re-emitted. |
| T-01-03 | Information disclosure | Permissions-Policy / header set | accept | Deny-by-default Permissions-Policy reduces feature exposure; no sensitive data in headers. |
| T-01-04 | Tampering | _headers drift from middleware constant | mitigate | gen-headers.mjs parity checker is wired into the `build` script (and build:cloudflare), which the pre-commit hook runs — drift fails the build automatically on every commit and CI build. |
| T-01-05 | Elevation/availability | Header misconfiguration breaks Keystatic OAuth or Formspree | mitigate | CSP ships Report-Only (cannot block resources); keystatic-scoped variant allows api.github.com; form-action allows formspree.io; Instagram embed hosts allowlisted. Live verification deferred to phase verification on a Cloudflare preview deploy (zero CSP-blocked-resource console errors required). |
| T-01-SC | Tampering | npm/pip/cargo installs | mitigate | No new package-manager installs in this plan (all code uses existing astro:middleware + Node stdlib). No legitimacy checkpoint required. |
